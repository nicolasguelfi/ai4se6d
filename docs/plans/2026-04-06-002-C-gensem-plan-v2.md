### PART 2 — L'évidence empirique (20 slides, 20 min)

**Contenu entièrement nouveau**. Le plan v1 avait 1 slide d'evidence. Ici c'est une section complète avec les 3 RCTs, le paradoxe, les surveys.

34. `bck_gensem_evidence_title` — "What Does the Evidence Say?"
35. `bck_gensem_evidence_peng` — **NOUVEAU** : RCT 1: Peng et al. 2023 — +55.8% sur HTTP server task, 95% CI: 21-89%. Limitation: single task.
36. `bck_gensem_evidence_cui` — **NOUVEAU** : RCT 2: Cui et al. 2024 — Le plus grand: 4,867 devs, 3 entreprises, 2-8 mois. +26.08% tasks completed. Juniors +27-39%.
37. `bck_gensem_evidence_metr` — **NOUVEAU** : RCT 3: METR 2025 — -19% slower! 16 experts, repos familiers. "Despite predicting 24% time reduction."
38. `bck_gensem_evidence_paradox` — **NOUVEAU** : Le Paradoxe de Productivité — tension Cui vs METR. Explications: prompt engineering overhead, disruption des mental models, context-switching.
39. `bck_gensem_evidence_perception` — **NOUVEAU** : Perception-Reality Gap — METR devs estimated +20%, actual -19%. Economists predicted 39%. Accuracy confidence Stack Overflow: 40%→29%.
40. `bck_gensem_evidence_surveys` — **NOUVEAU** : Tableau des 5 developer surveys (Stack Overflow 65K, JetBrains 24K, GitLab 3K, Pragmatic 2.5K, GitHub Octoverse). Cursor usage 17x YoY.
41. `bck_gensem_evidence_daniotti` — **NOUVEAU** : Daniotti 160K GitHub devs — seniors capturent TOUS les gains, juniors aucun bénéfice significatif. Contradicts conventional wisdom.
42. `bck_gensem_evidence_paradox_ai` — **NOUVEAU** : AI Paradox (GitLab): 7h/week lost. Faster coding creates bottlenecks elsewhere. 85% need platform engineering.
43. `bck_gensem_evidence_bain` — **NOUVEAU** : Enterprise data: Bain 10% tools alone → 25-30% with process transformation. McKinsey: redesign processes.
44. `bck_gensem_evidence_synthesis` — **NOUVEAU** : Synthèse en 3 points: (1) Gains réels mais variables, (2) Methodology is the multiplier, (3) Senior expertise + process = where real value is.
45-47. `bck_gensem_evidence_discussion` — Discussion: "Which finding surprises you most?" + Transition vers les frameworks
