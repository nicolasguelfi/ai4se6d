# Plan détaillé — Module ai4se6d_gensem v4

**Date**: 2026-04-14
**Cycle**: CE-GENSEM-004
**Statut**: En attente de validation GATE 1
**Module source (ancienne version)**: `modules/ai4se6d_gensem_v01/`
**Module cible (nouvelle version)**: `modules/ai4se6d_gensem/`

## Contexte

Module de 4 sessions × 3h (12h total) pour introduire, présenter et faire pratiquer la méthodologie GSE-One.
Les participants ont déjà vécu 2 jours de formation (modules genai_intro + vibecoding) incluant l'expérience du VibeCoding sur FreeSelfApp et la maîtrise de Cursor.

## Sources par ordre de priorité

1. **gse-one-spec.md** — Spécification complète de la méthodologie (16 principes, 4 LC, 23 commandes)
2. **gse-one-implementation-design.md** — Conception du plugin (architecture, agents, templates)
3. **gse-one/** — Fichiers d'implémentation (skills, agents, templates)
4. **sota-ai-4-se-EN.tex** — Fondation scientifique (sections 4-9, Appendix A)
5. **ai4se6d_gensem_v01/** — Blocs réutilisables de l'ancienne version du module

## Cas pratique : CalcApp

Toutes les pratiques utilisent CalcApp, une app de gestion de finances personnelles :

> *"I want to build CalcApp, a personal finance budgeting web app.*
> *Core features: Add expenses (amount, category, date) / List expenses filtered by month and category / Set monthly budgets per category / Show remaining budget per category with visual indicators"*

Repo : `~/Projets/AISE/ROS/trainings/AI4SE/githubs/calcapp` (branches v0-plans → v5-production)

## Design des slides

- **Minimaliste et graphique** : 1 idée = 1 slide, image d'illustration porteuse de sens
- **60-90 slides par bloc théorie** de 45 min
- **Texte minimal** : titre + 1-3 bullet points max, l'image porte le message
- **Code montré via screenshots ou extraits courts**, jamais de blocs de code denses

## P1 — Découverte libre de GSE-One (45 min)

**Objectif** : Vivre l'expérience GSE-One de bout en bout AVANT d'avoir les explications théoriques.

### Déroulement

| Min | Activité | Détail |
|-----|----------|--------|
| 0-5 | **Setup** | Distribuer le lien du repo GSE-One plugin pour Cursor. Installation : `gse install --target cursor` |
| 5-10 | **Consigne unique** | "Ouvrez CalcApp dans Cursor. Tapez `/gse:go`. Suivez le processus sans aide externe. Notez vos observations." |
| 10-40 | **Expérience libre** | Les participants suivent le flow GSE-One guidé par l'agent : HUG → COLLECT → ASSESS → PLAN → (début PRODUCE). Pas d'intervention du formateur. |
| 40-45 | **Debriefing express** | Tour de table rapide : qu'avez-vous compris ? qu'est-ce qui vous a surpris ? qu'est-ce qui vous a frustré ? |

### Points d'observation pour le debriefing (préparés pour le formateur)
- L'agent a-t-il posé des questions (HUG) ? → P4, P9
- A-t-il proposé des choix avec des conséquences (Gate) ? → P7, P8
- A-t-il refusé de faire quelque chose (guardrail) ? → P11
- A-t-il expliqué son niveau de confiance ? → P15
- A-t-il créé des branches/worktrees ? → P12
- Avez-vous eu l'impression de contrôler le processus ?

### CalcApp pour P1
- Branche de départ : `main` (code minimal ou v1-skeleton)
- Objectif suggéré par l'agent : "Ajouter la fonctionnalité de filtrage par mois"
- L'agent conduit le participant à travers le lifecycle complet
