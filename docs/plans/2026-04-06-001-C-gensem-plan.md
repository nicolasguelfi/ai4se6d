| Field | Value |
|-------|-------|
| **Title** | Design Philosophy |
| **Content** | **Foundational principle**: "Each unit of engineering work should make subsequent units easier — not harder." #### Slide 22 — `bck_gensem_ce_brainstorm_plan`

| Field | Value |
|-------|-------|
| **Title** | Phases 1-2: Brainstorm & Plan |
| **Content** | **Brainstorm** (`/ce:brainstorm`): Collaborative exploration BEFORE any code. #### Slide 23 — `bck_gensem_ce_work_review`

|-------|-------|
| **Title** | Phases 3-4: Work & Review |
| **Content** | **Work** (`/ce:work`): Executes the approved plan. Agent is constrained: deviations from plan require returning to Plan phase. **Review** (`/ce:review`): Multi-agent review from multiple perspectives: correctness, security, architecture, and learning extraction. Accepts: PR number, GitHub URL, branch name, markdown path, or current branch. Implements **n-version verification** — analogous to safety-critical systems engineering. |

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Agile approach for Part 2 | **Compound Engineering** | Only framework with concrete implementation (npm plugin), supports 10+ tools including Cursor (training tool), composable phases match agile principles, documented in SOTA Appendix A |
| Part 3 placement | **Day 6 PM** (not Day 3) | Participants need Days 3-5 experience before seeing the automated version. Day 6 "Advanced SE with Claude" slot is the natural home. |
| GenSEMOne tool constraint | **Cursor-only, no plugins** | Training tool is Cursor. Participants shouldn't need new tools for the mini-project. CE principles applied manually using Cursor's native features (rules, Plan/Act Mode, todos). |
| CalcApp v0.3 mapping | **CE phases to Day 3 workshops** | Makes the abstract framework concrete by mapping each CE phase to the practical activities participants will do that same day. |
| Slide count | **~63 slides** | Distributed across 3 days (~105 min total). Day 3: ~35 slides/60 min. Day 5: ~11 slides/30 min. Day 6: ~15 slides/60 min. Consistent with existing module density. |
